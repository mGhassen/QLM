# Rewrite desktop main entry

Replace `apps/desktop/src/main.tsx` so it bootstraps the apps/web router in pure SPA mode (no TanStack Start). The SPA mounts a `RouterProvider` into `<div id="root">`, wraps with the same `QueryClient` + provider stack as apps/web's web build.

## Done when

- [ ] `apps/desktop/src/main.tsx`:
  - Imports `routeTree` from `./routeTree.gen` (regenerated by task 001's plugin from apps/web routes).
  - Constructs `QueryClient` with the same defaults as `apps/web/src/router.tsx` (`staleTime: 60_000`).
  - Calls `createRouter({ routeTree, scrollRestoration: true, defaultPreload: 'intent', defaultPreloadStaleTime: 0, context: { queryClient } })`.
  - Wraps `<RouterProvider router={router} />` with `<RootProviders queryClient={queryClient}>` from `@/providers/root-providers` (the apps/web alias).
  - Mounts via `createRoot(document.getElementById('root')!).render(...)` inside `<StrictMode>`.
  - Re-uses the `declare module '@tanstack/react-router' { interface Register { router: typeof router } }` block.
- [ ] `apps/desktop/package.json` adds `@tanstack/react-query` dependency (catalog version) — the router context requires it.
- [ ] No imports from `@tanstack/react-start` or `@tanstack/react-start/client`.
- [ ] `pnpm typecheck` for `apps/desktop` passes (errors in apps/web routes that haven't been SPA-fixed surface here — fold those into task 003 if minor, or call them out for the curated-fallback path).

## Notes

- Spec anchor: `#75-presentation-appsweb`.
- `RootProviders` from apps/web pulls in i18n + Supabase auth + workspace context + theme + Sentry + telemetry. All of these need to work in SPA mode; some may need runtime gating. Treat any breakage that is NOT about `<Scripts/>`/`<HeadContent/>` (task 003 territory) as a real regression — leave the `useEffect`/provider intact and surface the issue.
- If `RootProviders` itself relies on Start-only features (`createServerFn`, etc.), this task's scope expands; flag in `## Questions surfaced` and consider a thinner desktop-specific provider stack as a follow-up.
