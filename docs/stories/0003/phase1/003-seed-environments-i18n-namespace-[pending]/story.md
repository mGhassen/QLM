# Seed environments i18n namespace

## Goal

Create the `environments` i18n namespace across every locale with the full key set from spec §11 (English filled with real copy, non-English as English-identical placeholders) so every component built in later stories can localise strings from day one without a second pass.

## Scope

**In scope**

- `packages/i18n/src/locales/en/environments.json` — every key from spec §11 (`environments.layer1.*`, `environments.layer2.*`, `environments.inspector.*`, `environments.status.*`, `environments.actions.*`, `environments.empty.*`, `environments.errors.*`, `environments.events.*`) with real English values.
- `packages/i18n/src/locales/{fr,es,de,…}/environments.json` — the same key structure in every existing locale. Values may be English-identical placeholders; translation to target languages is a separate workflow.
- Any namespace loader / index update the i18n package needs to discover the new `environments.json` namespace.
- Verify that the typed key union generated by `@guepard/i18n` includes `environments.*` so `t('environments.layer1.empty.heading')` is TypeScript-safe.

**Out of scope** (forces honest slicing)

- Translating non-English locales — placeholders are acceptable.
- Using any key in a component (→ Stories 005–009).
- Adding keys not listed in spec §11 (new keys get added by the story that needs them, with an explicit spec anchor reference).
- Changes to the i18n infrastructure itself (`react-i18next` wiring, `<Trans>` component).

## Acceptance criteria

- [ ] Every key listed in the spec §11 `environments.*` inventory exists in every locale file present in `packages/i18n/src/locales/`.
- [ ] `packages/i18n/src/locales/en/environments.json` has real, non-placeholder English strings for every key.
- [ ] Non-English locale files have matching key structure; values may be English-identical.
- [ ] A one-line Vitest test (`t('environments.layer1.empty.heading')` inside a fresh i18n instance) resolves to the English value without warnings.
- [ ] `pnpm typecheck` passes — the typed key union picks up the new namespace without manual cast.
- [ ] ESLint rule forbidding `react-i18next/Trans` (per `.claude/rules/i18n.md`) continues to pass on any touched files.
- [ ] No typos in key names (verified by a JSON-schema lint or simple string compare against the spec key list).
