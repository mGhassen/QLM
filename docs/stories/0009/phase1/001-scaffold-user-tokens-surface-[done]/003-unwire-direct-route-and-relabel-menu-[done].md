# Unwire direct route and relabel menu

## Purpose

Revert the task-002 work that wired the account menu directly to `/user/tokens` so the spec-amended (AM-1) architecture can land: delete the standalone route, remove `createUserTokensPath`, relabel the account-menu entry to "Settings", and swap `onAccessTokensClick` → `onSettingsClick` on both `UserProfileMenu` and `ShellUserProfileMenu`. The new handler is a **stub** (a `console.log` placeholder or an empty function) until Story 010 ships the real settings-dialog mount.

## Files

- `apps/web/src/routes/user/tokens.tsx` — **delete**. No standalone route in phase 1 per AM-1.
- `apps/web/src/config/paths.config.ts` — **remove** `createUserTokensPath()` export and the `user: { tokens: '/user/tokens' }` stanza. Phase 1 has no `/user/*` route.
- `apps/web/src/routeTree.gen.ts` — auto-regenerated by TanStack Router on next build. Verify the `/user/tokens` entry is gone after `pnpm web:dev` starts or `pnpm --filter web build`.
- `packages/ui/src/guepard/layout/user-profile-menu.tsx` — rename prop `onAccessTokensClick` → `onSettingsClick`. Also rename the **existing unused** `onSettingsClick` prop (for the gear icon) → `onProfileSettingsIconClick` to free the name. Update the `Key` icon import — if unused after rename, remove it; if kept for the new Settings button, keep the icon (use `Settings` or `SlidersHorizontal` from `lucide-react` per UX preference). Change the button label from "Access tokens" to "Settings" (localized via `tokens.menu.accessTokens` → `settings.menu.label` key — coordinate with Story 003 or temporarily use a literal with a `TODO(003)` marker).
- `packages/ui/src/guepard/shell/shell-user-profile-menu.tsx` — same rename + label changes as above.
- `apps/web/src/routes/organizations.tsx` — pass `onSettingsClick={() => console.log('TODO(story-010): open settings dialog')}` (or similar stub) in place of the old `onAccessTokensClick` handler. Remove the `createUserTokensPath` import — it no longer exists.
- `apps/web/src/shell/project-shell-host.tsx` — same stub + import cleanup.
