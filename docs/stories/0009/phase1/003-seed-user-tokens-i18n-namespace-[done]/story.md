# Seed user-tokens i18n namespace

## Goal

Create the `tokens` i18n namespace across every locale with the full key set from spec §11 (English filled with real copy, non-English as English-identical placeholders) so every component built in later stories can localise strings from day one and the placeholder heading from Story 001 gets replaced with a real localized string.

## Scope

**In scope**

- `packages/i18n/src/locales/en/tokens.json` — real English values for every key listed in spec §11 (menu, page, toolbar, table, status, scopes, generate dialog, reveal view, revoke dialog, empty state, errors).
- `packages/i18n/src/locales/{fr,es,de,…}/tokens.json` — same key structure in every existing locale, values may be English-identical placeholders.
- Any i18n namespace loader / index update the package needs to pick up the new `tokens.json`.
- Replace the `TODO(003)` fallback string in `apps/web/src/routes/user/tokens.tsx` (placed by Story 001) with `t('tokens.page.title')`.
- Verify the typed key union generated by `@guepard/i18n` includes `tokens.*`.

**Out of scope**

- Translations to actual non-English languages — placeholders are acceptable; translation is a separate workflow.
- Using any of these keys in a component (→ Stories 009–011).
- Adding keys not listed in spec §11 — new keys get added by the story that needs them, with an explicit spec anchor reference.
